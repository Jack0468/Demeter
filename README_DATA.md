# Demeter Data Ontology & Dataset Guide

This directory houses all raw datasets, processed training structures, and system outputs for the Demeter project. To maintain data integrity and prevent domain mismatch during model inference, datasets are strictly categorized by their biological and statistical function.

## 1. Raw Datasets (`data/raw/`)
Immutable source data. **Never modify these files directly.**

### Vision Stream (`raw/vision/`)
*   **PlantVillage**: Highly controlled, zoomed-in RGB images of individual leaves.
    *   *Function*: Used exclusively to train the primary CNN for discrete disease classification (e.g., Early Blight, Healthy). 
    *   *Limitation*: Contains no environmental context.
*   **Bellwether Snapshots**: Longitudinal images capturing the entirety of the plant (Setaria) over time.
    *   *Function*: Visual trajectory mapping and whole-plant stress detection.
*   **Dataset 3 (Manual Tiller Images)**: A specialized subset of 58 images linked to manual phenotypic measurements.
    *   *Function*: Training secondary computer vision models to regress continuous physical variables (e.g., tiller count, leaf angle) directly from pixels.

### Environmental Stream (`raw/environment/`)
*   **Danforth Growth Data (`danforth_growth.csv`)**: Tabular data detailing `Temperature`, `Soil_Moisture`, `Sunlight_Hours`, etc.
    *   *Function*: Trains the Random Forest Regressor to predict the continuous `Growth_Milestone` target based on environmental conditions.
*   **Bellwether Longitudinal (`bellwether_data.csv`)**: Time-series tabular data tracking `weight before`, `water amount`, and `weight after`.
    *   *Function*: Used for legacy water-stress analysis and biomass delta predictions.
*   **Manual Measurements (`manual_measurements.txt`)**: Tabular phenotypic data linked to Dataset 3 images.

## 2. Processed Data (`data/processed/`)
Data generated by scripts (e.g., `scripts/data_prep/bootstrap_svm_data.py`) or pre-split test sets.

*   **`svm_training/svm_bootstrap_data.csv`**: A synthetic dataset created by cross-sampling real CNN visual confidences and real RF environmental predictions. 
    *   *Warning*: Because PlantVillage and Danforth data lack a shared intersection, this data assumes visual disease and environmental health are statistically independent. Use carefully for the SVM Status Engine.

## 3. System Outputs (`data/outputs/`)
Real-time artifacts generated by `inference_engine.py` and `status_engine.py` meant for the web dashboard.

*   **`latest_diagnosis.json`**: The most recent unified prediction payload.
*   **`diagnosis_history.json`**: A rolling JSON array of recent predictions used by the API for historical graphing.

## 4. System Logs (`data/logs/`)
Historical tabular tracking of the system's runtime predictions.

*   **`inference_logs.csv`**: The unified logging destination for all system inferences. Used to evaluate real-world model drift and provide actual historical distributions to mix with bootstrap data.

## 5. Cache (`data/cache/`)
Temporary files, such as `metadata_cache.pkl`, generated by TensorFlow dataset loaders or Pandas merges to speed up sequential pipeline runs.
